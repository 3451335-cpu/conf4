Конвертер конфигурационного языка 
1. Общее описание проекта
Этот репозиторий содержит решение для курсового задания по разработке конвертера, который переводит конфигурационный файл, написанный на учебном конфигурационном языке, в общепринятый формат JSON.

Основная работа конвертера лежит в двух этапах:

Парсинг: С помощью библиотеки Lark и LALR-алгоритма мы строим Абстрактное Синтаксическое Дерево (AST).

Трансляция: Специализированный Трансформер выполняет обход этого дерева. На этом этапе происходит подстановка значений констант и формирование структуры данных.

В конечном итоге, преобразованная структура данных сериализуется в формат JSON с использованием стандартной библиотеки Python.

Синтаксис нашего DSL 
Наш язык поддерживает следующие базовые конструкции:

Константы: объявляются с помощью var, например: var PORT 8080.0.

Данные: организованы в словари ([ключ : значение]) и массивы ({ значение. значение. ... }).

Подстановка констант: значения констант подставляются с помощью $имя$, например: $PORT$.

2. Архитектура и описание компонентов
Вся логика сосредоточена в файле main.py. Проект организован вокруг трех ключевых компонентов.

2.1. Грамматика (GRAMMAR)
Здесь определены все правила языка. Грамматика описывает, как выглядят наши структуры:

Декларация констант (const_decl) и их использование через константные ссылки (const_ref).

Структурные элементы: словари (dict) с разделителем : и массивы (array) с разделителем ..

Базовые типы: числа с плавающей точкой (NUMBER) и строки в формате q(...) (STRING).

Поддерживаются многострочные комментарии (/* ... */).

2.2. Класс MyTransformer
Этот класс, наследующий функционал Lark, отвечает за перевод узлов AST в реальные Python-объекты.

Управление константами: Метод const_decl наполняет внутренний словарь self.consts, доступный для всего парсинга.

Подстановка значений: Метод const_ref заменяет ссылки на константы ($имя$) на их актуальные значения.

Построение структур: Методы dict, array, dict_items формируют словари и списки Python из элементов AST.

2.3. Класс Converter
Это точка входа, которая объединяет парсер и трансформер. Класс отвечает за:

Чтение входного файла (parse_file)

Парсинг и преобразование контента (parse_content)

Генерацию JSON-выхода (to_json) с помощью стандартной библиотеки json

3. Разработка и запуск
3.1. Установка и зависимости
Для работы проекта необходима библиотека Lark. Все зависимости указаны в файле requirements.txt.
# Устанавливаем все, что нужно, одной командой
pip install -r requirements.txt
3.2. Запуск конвертера
Запуск происходит через командную строку. Необходимо указать путь к входному файлу (.conf).
# Пример
python main.py -i example1.conf
3.3. Организация и запуск функциональных тестов
Для обеспечения надежности и демонстрации покрытия всех конструкций языка используется отдельный модуль для тестирования — test_converter.py.

Команда для запуска тестов
python -m unittest test.py
